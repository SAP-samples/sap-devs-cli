AI Platform Adapter System

How the tool injects SAP context into each AI tool. Adding a new tool = adding one YAML file.

📄 content/adapters/claude-code.yaml
id: claude-code
name: Claude Code
type: file-inject
 
# Where to write context
targets:
- scope: global
path: "~/.claude/CLAUDE.md"
mode: append-section
section: "SAP Developer Context"
- scope: project
path: "./CLAUDE.md"
mode: append-section
section: "SAP Developer Context"
 
# MCP server wiring
mcp_config:
path: "~/.claude/settings.json"
format: json
key: "mcpServers"
 
# Detection — is this tool installed?
detect:
- command: "claude --version"
- path: "~/.claude"
Adapter types + known tools
FILE INJECT — writes to a config file
✓ claude-code → ~/.claude/CLAUDE.md
✓ cursor → .cursorrules
✓ copilot → .github/copilot-instructions.md
✓ continue → ~/.continue/config.json
✓ jetbrains-ai → .idea/ai-instructions.md
✓ cody → .cody/context.md
CLIPBOARD EXPORT — copies formatted text
✓ chatgpt → custom instructions format
✓ gemini → system prompt format
✓ claude-ai → project instructions format
✓ copilot-chat → custom format
MCP WIRE — installs + configures MCP server
✓ claude-code → ~/.claude/settings.json
✓ cursor → ~/.cursor/mcp.json
✓ vscode-mcp → settings.json
💡 Adding "tool X tomorrow" = drop a new .yaml file into content/adapters/ and submit a PR. No Go code needed.

Three adapter types cover all cases: file injection for local coding tools, clipboard export for web AI tools, and MCP wiring for hosts that support it. Does this adapter model make sense?
