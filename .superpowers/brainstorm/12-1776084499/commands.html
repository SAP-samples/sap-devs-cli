CLI Command Structure

The primary commands. Does this cover the workflows you have in mind?

Command Tree
sap-devs
init           # First-time setup wizard
sync           # Pull latest from all content repos
inject         # Push SAP context to AI tools
--tool <name>  # target one tool
--global       # user-level (default)
--project      # current repo only
--dry-run      # preview changes
profile
set <name>     # set developer persona
show           # show current profile
list           # list available profiles
mcp
list           # available MCP servers
install <id>   # install + wire one server
install --all  # install all for profile
status         # show what's installed
tip             # print a tip (shell profile hook)
resources
list           # browse curated resources
open <id>      # open in browser
search <query> # full-text search
config
show           # show current config
set <k> <v>    # set a config value
company <url>  # point to company repo
update         # self-update binary
Key Workflows
FIRST TIME SETUP
$ sap-devs init
✓ Downloads content cache
✓ Asks: what kind of developer are you?
✓ Asks: which AI tools do you use?
✓ Injects SAP context into each tool
✓ Suggests MCP servers to install
✓ Adds tip hook to shell profile
DAILY TERMINAL OPEN
$ sap-devs tip
💡 CAP tip: Use cds.ql for type-safe
    CQL queries instead of raw strings.
    → cap.cloud.sap/docs/node.js/cds-ql
START A NEW PROJECT
$ cd my-cap-project
$ sap-devs inject --project
✓ Writes ./.sap-devs/context.yaml
✓ Injects into ./CLAUDE.md
✓ Injects into ./.cursorrules
✓ Injects into ./.github/copilot-instructions.md
STAY CURRENT
$ sap-devs sync && sap-devs inject
✓ Pulls latest from official + company repos
✓ Re-injects updated context to all tools
