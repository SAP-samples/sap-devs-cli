Environment Doctor

sap-devs doctor — checks your local tool installations against requirements defined in the content repo. Profile-aware, cross-platform.

$ sap-devs doctor
Checking environment for profile: cap-developer
✓ Node.js      v22.1.0   (required: ≥18)
✓ npm         10.7.0    (required: ≥8)
✓ @sap/cds     8.2.1     (required: ≥7)
⚠ Java         11.0.2    (required: ≥17) — upgrade needed
✗ BTP CLI      not found
✗ Cloud MTA Tool not found
✓ Git          2.44.0
─────────────────────────────────────
optional (not required for cap-developer):
○ ABAP SDK   (install: sap-devs mcp install abap)
2 issues found. Run sap-devs doctor --fix for install commands.
📄 content/packs/cap/tools.yaml (extended)
- id: nodejs
name: Node.js
required: ">=18.0.0"
detect:
command: "node --version"
pattern: "v(\\d+\\.\\d+\\.\\d+)"
install:
windows: "winget install OpenJS.NodeJS.LTS"
macos: "brew install node@20"
linux: "nvm install 20"
docs: "https://nodejs.org"
 
- id: cds-cli
name: SAP CDS CLI
required: ">=7.0.0"
detect:
command: "cds --version"
pattern: "@sap/cds: (\\d+\\.\\d+\\.\\d+)"
install:
all: "npm install -g @sap/cds-dk"
docs: "https://cap.cloud.sap"

Detection is a regex pattern over command output — fully cross-platform, no OS-specific code in the binary. Install commands can be per-platform (windows/macos/linux) or a single all when the command is the same everywhere. --fix prints the commands; it never runs them without explicit consent.
