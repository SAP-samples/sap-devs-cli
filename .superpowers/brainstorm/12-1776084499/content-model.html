📄 content/packs/cap/pack.yaml
id: cap
name: SAP Cloud Application Programming Model
description: CAP framework for Node.js and Java
tags: [cloud, btp, node, java, odata]
profiles: [cap-developer, btp-developer]
weight: 100 # base priority
 
# context.md injected into AI tools
# resources.yaml — curated links:
- title: CAP Documentation
  url: https://cap.cloud.sap
  type: official-docs
  tags: [reference]
- title: SAP Devs YouTube — CAP
  url: https://youtube.com/...
  type: video
  advocate: daniel-van-leeuwen
 
# mcp.yaml — MCP server:
- id: cap-mcp-server
  install: npx @sap/cap-mcp
  hosts: [claude-code, cursor]

A profile (e.g., abap-developer.yaml) lists which packs to weight highly and in what order — it doesn't restrict access, just sets priority. The adapter files define how each AI tool gets content written to it (file paths, format, template). Adding support for a new AI tool = adding one YAML file.

Does this structure make sense? Any pack domains missing from the initial list?
